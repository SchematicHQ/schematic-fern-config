---
title: The Plan object
---

The `Plan` object describes plans in your application.

`Plans` can be associated with `Companies` based on plan audiences.

### Attributes

<ParamField path="id" type="string">
Unique `id` generated by Schematic for the object.
</ParamField>

<ParamField path="name" type="string">
The plan’s name, which is displayable to Schematic users.
</ParamField>

<ParamField path="created_at" type="datetime">
Datetime when the plan was created. Format is ISO 8601.
</ParamField>

<ParamField path="updated_at" type="datetime">
Datetime of last update to plan data. Format is ISO 8601.
</ParamField>

</Aside>

### Plan versions

Every plan is versioned. Each version has its own entitlements and pricing, and a `status`:

- `published` — the live version, used for new checkouts and plan assignments
- `draft` — a work-in-progress version that is not yet live
- `archived` — a previous version that is no longer the live version

A version is represented by the `PlanVersion` object, whose key attributes are `version` (an incrementing integer) and `status` (one of the values above). Detailed plan responses (for example, the plan detail endpoint) include the plan's current `active_version` and, when one exists, its `draft_version`.

Companies reference the specific plan version they are on, so different companies can be on different versions of the same plan. See the [Plan Versions](/catalog/plans#plan-versions) documentation for the full model.
